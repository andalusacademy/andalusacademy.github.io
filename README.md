# نظام إدارة مركز الاندلس للتدريب

نظام إدارة أكاديمي متكامل لـ **مركز الاندلس للتدريب** — ملف HTML واحد (Single File App) مربوط بـ Supabase كقاعدة بيانات سحابية، بدون أي Backend منفصل. يغطي إدارة الطلاب، البرامج، الحضور بالمحاضرات، الدرجات، الامتحانات الإلكترونية، المصروفات، الحجوزات، المهام، الإشعارات، التنبؤ بالمخاطر، والشهادات.

> **الإصدار الحالي: v3.6** — يشمل الواجهة الجديدة (شريط علوي + بطاقات)، نظام المصروفات المُعاد هيكلته، الحضور المرتبط بالمحاضرات، ونظام الامتحانات الإلكترونية الكامل.

## 🗄️ قاعدة البيانات (Supabase)

### الإعداد الأول
1. ادخل على [supabase.com](https://supabase.com) وافتح المشروع `uvwarprsghwvrfmdaaqv`.
2. روح على **SQL Editor**.
3. نفّذ ملفات الـ SQL **بالترتيب الزمني** الموجودة فى فولدر `/sql`:

| الترتيب | الملف | الوصف |
|---|---|---|
| 1 | `andalus_v3_1_sql_setup.sql` | الإعداد الأساسي للجداول |
| 2 | `andalus_v3_2_sql_setup.sql` | الرقم القومي، وقت استحقاق المهام |
| 3 | `andalus_v3_3_sql_setup.sql` | إجمالي المستحق اليدوي، session_id للحضور، أعمدة المحاضرات والمدفوعات |
| 4 | `andalus_v3_4_fix.sql` | تصحيح UNIQUE CONSTRAINT، نوع عمود program_id، تخفيف قيد recorded_by |
| 5 | `andalus_v3_5_exams.sql` | جداول الامتحانات الإلكترونية (exams, exam_questions, exam_attempts) |
| 6 | `andalus_v3_6_exam_enhancements.sql` | الدرجة الكلية المستقلة، كشف الغش، ملاحظات الدرجات |
| — | `andalus_exam_login_fix.sql` | تعطيل RLS على students/grades (مطلوب لعمل بوابة الطالب) |

4. نفّذ كل بلوك لوحده، وتجاهل أي خطأ بمعنى **"already exists"** — ده طبيعي لو الجدول/العمود اتعمل قبل كده.

> ⚠️ **مهم:** جدولا `students` و `grades` يجب أن يكون فيهم **Row Level Security (RLS) معطّلة**، لأن بوابة الامتحانات الإلكترونية للطلاب تتصل بدون تسجيل دخول Supabase Auth. تأكد من تنفيذ `andalus_exam_login_fix.sql`.

## ✨ المميزات الكاملة

### 🖥️ الواجهة العامة
- شريط علوي ثابت (لوجو، حاسبة، تصدير Excel، إشعارات، تسجيل خروج، بيانات المستخدم) بدل القائمة الجانبية القديمة
- صفحة رئيسية ببطاقات ملوّنة كبيرة للتنقل بين كل أقسام النظام، بنفس ترتيب الأقسام الإدارية
- فلتر فروع تفاعلي (الكل / كفر الدوار / فيكتوريا) يُحدّث كل الإحصائيات والرسوم البيانية فورًا

### إدارة الطلاب
- إضافة / تعديل / حذف بيانات الطلاب الكاملة، شاملة الرقم القومي وإجمالي المستحق اليدوي
- رفع صورة شخصية + حذف جماعي + استيراد دفعة طلاب من Excel
- بحث وفلترة بالاسم / رقم القيد / الفرع / البرنامج
- **سجل تعديلات (Audit Trail)**: تاب فى بروفايل كل طالب يوضح كل تعديل حصل على بياناته، بالتفصيل (القيمة القديمة والجديدة)، مع التاريخ والمستخدم

### البرامج والكورسات
- إضافة برنامج بنظام واحد (كورس) أو عدة أنظمة (برنامج)، وتعديل أي برنامج موجود
- أي برنامج جديد يظهر تلقائيًا فى كل القوائم المرتبطة

### الحضور والغياب (مُعاد بناؤه بالكامل)
- صفحة مستقلة: اختيار محاضرة من القائمة (مش برنامج وتاريخ عام)، تظهر كل طلابها
- زر حضور / غياب فوري لكل طالب يُحفظ مباشرة، مرتبط بـ (الطالب + المحاضرة + التاريخ) لتفادي تعارض محاضرتين فى يوم واحد
- عدادات حضور/غياب فورية + تصدير الحضور والغياب كل واحد فى ملف Excel منفصل
- زر واتساب تلقائي عند تسجيل غياب: يتضمن اسم المادة والتاريخ، وتحذير بالحرمان من استكمال البرنامج عند تجاوز 25% غياب
- نسبة الحضور والغياب لكل مادة تظهر فى بروفايل الطالب مع شريط تحذيري ملوّن (أخضر/أصفر/أحمر)

### الدرجات والمواد
- درجات لكل مادة (دور أول / دور تاني) مع حساب ناجح/راسب تلقائي
- تمييز تلقائي للدرجات القادمة من امتحان إلكتروني، مع ملاحظات (مثل "غشاش")
- لوحة الأوائل العامة: ترتيب كل الطلاب من كل الفروع بمتوسط درجاتهم الإجمالي

### 💻 الامتحانات الإلكترونية (ميزة جديدة بالكامل)
- **للموظف/المدير**: إنشاء امتحان محدد بمادة + برنامج + نظام + فرع، بمدة زمنية وفترة توفر ودرجة كلية مستقلة يكتبها بنفسه
- إضافة أسئلة (اختيار من متعدد أو صح/غلط) بدرجة مستقلة لكل سؤال
- إنشاء "دور ثاني سهل" بضغطة واحدة: نسخة كاملة من الامتحان قابلة للتسهيل
- **للطالب**: بوابة دخول مستقلة (`?exam-portal=1`) برقم القيد كيوزر نيم وباسورد
- ترتيب أسئلة عشوائي مختلف لكل طالب، تايمر تنازلي محفوظ بالسيرفر (لا يتأثر بإعادة تحميل الصفحة)
- **كشف غش صارم**: أي خروج من صفحة الامتحان (تبديل تبويب، تصغير) يُنهي الامتحان فورًا بدرجة صفر، مع تفرقة دقيقة بين الغش وإعادة تحميل الصفحة العادية
- تسليم تلقائي عند انتهاء الوقت، وتصحيح فوري بدون تدخل بشري
- النتيجة تُسجَّل تلقائيًا فى تاب "الدرجات" الرسمي بعلامة "💻 إلكتروني"
- **تاب نتائج شامل**: يعرض كل طلاب الفصل المطابقين (مش بس مَن امتحن)، ومَن لم يحضر يُسجَّل راسبًا تلقائيًا
- **صفحة "🎯 درجات الامتحانات"**: تابات (الكل/ورقي/إلكتروني) + فلتر بالمادة والفرع لمراجعة كل النتائج فى مكان واحد

### المصروفات والأقساط (مُعاد هيكلته بالكامل)
- **إجمالي المستحق**: رقم يدوي يُكتب مرة واحدة فى بيانات الطالب عند التسجيل (غير مرتبط بنظام ثابت)
- **إجمالي المدفوع**: مجموع كل البنود المدفوعة فعليًا، **المتبقي = المستحق − المدفوع**
- **رسوم مستقلة بالكامل عن المعادلة**: شنطة الأدوات، فتح الملف، وكورس الإنجليزي — تُعرض وتُدفع بشكل مستقل تمامًا ولا تدخل فى أي حساب
- امتحان تحديد المستوى (إنجليزي): نتيجة "اجتاز / لم يجتاز" بسيطة بدل الدرجات
- زر تعديل + زر واتساب لكل بند دفعة (رسالة تتضمن اسم البند وتاريخ اليوم)
- غرامة تأخير تلقائية بعد يوم 5 من الشهر
- تقرير مالي شامل + **توقع الإيرادات الشهرية القادمة** بناءً على الأقساط المسجَّلة فعليًا لكل الطلاب
- تصدير Excel لكل الجداول

### الإجراءات
- تسجيل إجراءات إدارية (تأخر مصاريف، استدعاء، إنذار، فصل) مع واتساب تلقائي لولي الأمر

### الحجوزات والاستفسارات (Leads)
- تسجيل حجز/استفسار جديد مع رسالة واتساب تلقائية "تم استلام طلبكم"
- **زر "📋 المستندات المطلوبة"**: رسالة واتساب جاهزة بكل المستندات والرسوم المطلوبة للتقديم، تُرسل قبل التحويل لطالب
- زر "تحويل لطالب": يفتح فورم الطالب مع تعبئة البيانات تلقائيًا، وتحديث حالة الحجز + رسالة قبول تلقائية

### المهام والإشعارات
- إضافة مهام بتاريخ ووقت استحقاق محدد، مع بوب أب تذكير قابل للإلغاء
- جرس إشعارات يجمع: مدفوعات متأخرة، نسبة غياب عالية، مهام مستحقة، **وأقساط مستحقة قريبًا (خلال 3 أيام) قبل فوات الموعد**

### ⚠️ التنبؤ بالمخاطر ولوحة المعلومات
- **قسم "طلاب معرّضون للخطر"** فى الرئيسية: درجة خطر محسوبة من نسبة الغياب + تأخر الدفع + الدرجات الضعيفة + وجود إجراء إداري (إنذار/فصل يرفع الخطر بشكل ملحوظ)
- يظهر بس للطلاب اللي خطرهم فوق الحد الأدنى، مرتبين من الأخطر للأقل، مع توضيح أسباب الخطر بالتفصيل

### الشهادات
- بناء قالب شهادة من صورة مرفوعة، سحب اسم الطالب وتحديد مكانه، طباعة شهادة مباشرة لكل طالب

### النظام العام
- تسجيل دخول/خروج عبر Supabase Auth (سحابي، من أي جهاز) — منفصل تمامًا عن بوابة الامتحانات (التي لا تستخدم Auth)
- صلاحيات مدير / موظف، وربط كل موظف بفرع
- سجل نشاط كامل لكل العمليات فى النظام
- حاسبة وتقويم ميلادي/هجري فى الواجهة الرئيسية
- رسائل واتساب جماعية (Bulk WhatsApp)
- حماية من XSS وفحوصات أساسية على المدخلات (هاتف مصري، ID آمن)
- حماية ثلاثية الطبقات ضد تكرار البيانات (race conditions) عند الإضافة، باستخدام UNIQUE CONSTRAINTs حقيقية (لا CREATE UNIQUE INDEX وحدها) لضمان عمل upsert بشكل صحيح مع PostgREST

## 🔐 ملاحظة أمان مهمة: بوابة الامتحانات

بوابة الامتحانات الإلكترونية للطلاب **لا تستخدم Supabase Auth** — الطالب يدخل برقم القيد كيوزر نيم وباسورد، والنظام يتحقق مباشرة من جدول `students` باستخدام نفس مفتاح `anon key` العام المستخدم فى باقي النظام. لهذا:

- جدولا `students` و `grades` يجب أن تكون فيهم RLS معطّلة (تمامًا مثل باقي جداول النظام: `payments`, `attendance`, `exams`...).
- الإجابات الصحيحة للأسئلة (`correct_answer`) **لا تُرسل للمتصفح أبدًا** إلا بعد ما الطالب يسلّم الامتحان فعليًا — يتم جلبها فى استعلام منفصل وقت التسليم فقط، لتقليل فرصة الغش عبر فحص كود الصفحة.
- هذا المستوى من الحماية مناسب لسياق امتحانات داخلية لمركز تدريب، وليس مصمَّمًا لمنافسات أو امتحانات رسمية معتمدة تتطلب تأمينًا على مستوى Enterprise.

## 📝 ملاحظات للمطور

- أي تعديل على هيكل قاعدة البيانات (عمود/جدول جديد) لازم يترفق بملف SQL منفصل مرقّم (v3.7, v3.8...) بدل تعديل الملفات القديمة.
- الكود كله فى ملف واحد لتسهيل النشر على GitHub Pages بدون build process.
- لو ظهر خطأ `42703` أو `PGRST204` فى أي عملية حفظ، غالبًا معناه إن عمود فى قاعدة البيانات ناقص — راجع آخر ملف SQL وتأكد إنه اتنفذ بالكامل.
- لو ظهر خطأ `"there is no unique or exclusion constraint matching the ON CONFLICT specification"` عند استخدام `upsert`، فهذا يعني إن العمود محتاج `UNIQUE CONSTRAINT` حقيقي (لا يكفي `CREATE UNIQUE INDEX` بمفرده) — راجع نمط الحل فى `andalus_v3_4_fix.sql`.
- لو ظهرت رسالة `"violates foreign key constraint"` فى عملية من بوابة الامتحانات، تأكد من أن الحقول الاختيارية (مثل `recorded_by`) تسمح بـ `NULL`.
